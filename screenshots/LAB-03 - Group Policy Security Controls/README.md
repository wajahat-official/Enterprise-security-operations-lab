# LAB-03 – Group Policy Security Controls

## Overview

This lab focused on implementing security controls using Group Policy Objects (GPOs) within an Active Directory environment. Group Policy was used to enforce centralized security settings and establish baseline security configurations across domain-joined systems.

## Objectives

* Create and manage Group Policy Objects (GPOs)
* Implement password security policies
* Configure account lockout settings
* Apply security configurations to domain users and computers
* Validate policy deployment and enforcement

## Technologies Used

* Active Directory Domain Services (AD DS)
* Group Policy Management Console (GPMC)
* Windows Server
* Domain Users & Computers
* Security Policies

## Tasks Performed

### 1. Group Policy Management

Accessed and configured Group Policy Management Console (GPMC) to manage domain security policies.

### 2. Password Policy Configuration

Configured domain-wide password requirements including:

* Minimum password length
* Password complexity requirements
* Password history
* Password expiration settings

### 3. Account Lockout Policy

Implemented account lockout protections to mitigate brute-force login attempts.

Configured:

* Account lockout threshold
* Lockout duration
* Reset lockout counter settings

### 4. Policy Linking

Linked security policies to the appropriate Organizational Units (OUs) and domain containers.

### 5. Policy Deployment Verification

Forced policy updates and verified successful application on client systems using policy validation tools.
