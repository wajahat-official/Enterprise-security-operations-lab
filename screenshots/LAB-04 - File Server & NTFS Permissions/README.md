# LAB-04 – File Server & NTFS Permissions

## Overview

This lab focused on implementing file-sharing services and access control mechanisms using NTFS permissions within a Windows Server environment. The objective was to secure shared resources by applying the principle of least privilege and validating user access based on assigned permissions.

## Objectives

* Configure shared folders on a Windows Server
* Implement NTFS permissions
* Assign access rights to users and groups
* Validate file access controls
* Enforce least-privilege security practices

## Technologies Used

* Windows Server
* Active Directory Domain Services (AD DS)
* NTFS Permissions
* Shared Folders
* Security Groups
* File Server Resource Management

## Tasks Performed

### 1. File Server Configuration

Configured a dedicated file-sharing location on the Windows Server to host organizational resources.

### 2. Shared Folder Creation

Created shared folders and configured share permissions for domain users and groups.

### 3. NTFS Permission Assignment

Applied NTFS permissions to control access to files and directories based on job roles and security requirements.

### 4. Security Group Integration

Used Active Directory security groups to simplify permission management and improve scalability.

### 5. Access Validation

Tested user access to verify that permissions were correctly enforced and unauthorized access was prevented.

## Key Skills Demonstrated
