# LAB-02 – Domain Join & DNS Verification

## Overview

This lab focused on integrating a Windows client system into the Active Directory domain and validating DNS functionality within the enterprise environment.

## Objectives

* Join a Windows client to the Active Directory domain
* Verify domain authentication
* Validate DNS name resolution
* Test communication between client and Domain Controller
* Confirm successful domain membership

## Technologies Used

* Windows Server
* Active Directory Domain Services (AD DS)
* DNS
* Windows Client
* Domain Authentication

## Tasks Performed

### 1. Client Network Configuration

Configured the Windows client with the appropriate IP addressing and DNS settings required for domain communication.

### 2. Domain Join Process

Successfully joined the Windows client machine to the Active Directory domain.

### 3. Authentication Verification

Logged into the client using domain user credentials to verify Active Directory authentication.

### 4. DNS Validation

Performed DNS resolution tests to confirm proper communication between the client and Domain Controller.

### 5. Connectivity Testing

Verified network connectivity between enterprise systems using hostname and IP-based communication.

## Key Skills Demonstrated

* Active Directory Integration
* Windows Client Administration
* Domain Authentication
* DNS Troubleshooting
* Enterprise Network Configuration

## Validation Results

The following checks were successfully completed:

* Client joined to Active Directory domain
* Domain user authentication verified
* DNS name resolution functioning correctly
* Communication with Domain Controller confirmed
* Enterprise connectivity validated

## Outcome

Successfully integrated a Windows endpoint into the enterprise Active Directory environment, establishing centralized authentication and management capabilities required for future security monitoring and administration tasks.
