# LAB-08 – Splunk SIEM Deployment

## Overview

This lab focused on deploying and configuring Splunk Enterprise SIEM to centralize log collection, analysis, and security monitoring activities within the enterprise-style lab environment.

## Objectives

* Install Splunk Enterprise
* Configure Splunk services
* Enable log ingestion
* Deploy Universal Forwarder
* Validate centralized log collection

## Technologies Used

* Ubuntu Server
* Splunk Enterprise
* Splunk Universal Forwarder
* Windows Security Logs
* Windows Event Forwarding

## Tasks Performed

### 1. Ubuntu Server Deployment

Installed and configured Ubuntu Server for Splunk deployment.

### 2. Splunk Enterprise Installation

Installed and configured Splunk Enterprise.

### 3. Receiving Port Configuration

Enabled Splunk receiving port 9997.

### 4. Universal Forwarder Deployment

Installed and configured Splunk Universal Forwarder.

### 5. Log Ingestion Validation

Verified Windows Security Events were successfully ingested into Splunk.

### 6. Dashboard Development

Created monitoring dashboards and performed security event analysis.

## Key Skills Demonstrated

* SIEM Administration
* Splunk Enterprise Deployment
* Universal Forwarder Configuration
* Log Management
* Security Monitoring

## Outcome

Successfully deployed Splunk Enterprise SIEM and centralized Windows Security Event collection for monitoring and analysis.
