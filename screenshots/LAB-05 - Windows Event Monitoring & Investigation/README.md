# LAB-05 – Windows Event Monitoring

## Overview

This lab focused on monitoring and analyzing Windows Security Events generated within the Active Directory environment. The objective was to understand how Windows logs security-related activities and how these events can be used for threat detection, auditing, and incident investigation.

## Objectives

* Generate Windows security events
* Monitor Security Event Logs
* Identify authentication-related activities
* Analyze Windows Event IDs
* Establish a foundation for centralized logging and SIEM integration

## Technologies Used

* Windows Server
* Windows Event Viewer
* Active Directory
* Security Logs
* Windows Security Auditing

## Tasks Performed

### 1. Security Event Log Review

Accessed Windows Event Viewer and reviewed Security Logs generated by the operating system and Active Directory services.

### 2. Authentication Event Monitoring

Monitored user authentication activities including:

* Successful logons
* Failed logon attempts
* Account authentication events

### 3. Security Event Identification

Identified important Windows Security Event IDs relevant to security monitoring and threat detection.

### 4. Event Analysis

Analyzed event details including:

* User accounts
* Source systems
* Authentication methods
* Event timestamps
* Event outcomes

### 5. Security Audit Validation

Verified that security auditing was functioning correctly and generating relevant security events.

## Key Event IDs Observed

### Authentication Events

* Event ID 4624 – Successful Logon
* Event ID 4625 – Failed Logon
* Event ID 4634 – Logoff Event

### Account Management Events

* Event ID 4720 – User Account Created
* Event ID 4728 – User Added to Security Group
* Event ID 4732 – Member Added to Local Group
